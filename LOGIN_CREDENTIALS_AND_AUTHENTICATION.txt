================================================================================
API AUTHENTICATION HEADER
================================================================================

When making API calls, include the JWT token in the header:

Format:
  Authorization: Bearer YOUR_JWT_TOKEN_HERE

Example:
  curl -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..." \
       http://localhost:8000/api/v1/agents

Getting Your Token:

1. Login Endpoint:
   POST /api/v1/auth/login
   Body:
   {
     "username": "admin",
     "password": "Admin@12345"
   }

2. Response:
   {
     "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
     "refresh_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
     "token_type": "bearer",
     "expires_in": 900
   }

3. Use access_token in Authorization header

SECURITY BEST PRACTICES
================================================================================

DO:
  ✓ Use strong passwords (8+ chars, uppercase, lowercase, numbers, symbols)
  ✓ Change password regularly (every 90 days)
  ✓ Enable MFA for admin accounts
  ✓ Use unique usernames
  ✓ Log out after use
  ✓ Keep JWT secret key secret
  ✓ Use HTTPS in production
  ✓ Rotate API keys monthly

DON'T:

  ✗ Use simple passwords (123456, password)
  ✗ Commit .env file to git
  ✗ Use default credentials in production
  ✗ Log with sensitive data
  ✗ Store passwords in plain text
  ✗ Use HTTP in production (use HTTPS)
  ✗ Reuse passwords across accounts

================================================================================
TROUBLESHOOTING LOGIN ISSUES
================================================================================

PROBLEM 1: "Invalid username or password"
  Solution:
  - Check spelling (case-sensitive)
  - Verify caps lock is off
  - Reset password using "Forgot Password"
  - Try default test credentials first

PROBLEM 2: "Account locked - too many login attempts"
  Solution:
  - Wait 15 minutes (lockout duration)
  - Or contact admin to unlock
  - Or check .env for LOCKOUT_DURATION_MINUTES

PROBLEM 3: "Token expired"
  Solution:
  - Login again (new token will be issued)
  - Tokens expire after 15 minutes
  - Use refresh token to get new access token

PROBLEM 4: "Access denied"
  Solution:
  - Check your user role
  - Admin has full access
  - User/Viewer roles have limited access
  - Ask admin to upgrade your permissions

PROBLEM 5: "MFA code not working"
  Solution:
  - Verify code is 6 digits
  - Check time on your phone/authenticator
  - Time must be synchronized
  - Use backup code if available
  - Regenerate MFA if needed

PRODUCTION SECURITY SETTINGS
================================================================================

Before deploying to production:

CRITICAL CHANGES:

1. Change JWT Secret Key
   JWT_SECRET_KEY=your-very-long-random-secret-string-here

2. Disable Test Accounts
   TEST_ACCOUNTS_ENABLED=false

3. Enable HTTPS
   HTTPS_ENABLED=true
   SSL_CERTIFICATE=/path/to/cert.pem
   SSL_KEY=/path/to/key.pem

4. Set Strong Password Requirements
   PASSWORD_MIN_LENGTH=12
   PASSWORD_REQUIRE_UPPERCASE=true
   PASSWORD_REQUIRE_NUMBERS=true
   PASSWORD_REQUIRE_SPECIAL_CHARS=true

5. Enable Rate Limiting
   RATE_LIMIT_ENABLED=true
   RATE_LIMIT_REQUESTS=100
   RATE_LIMIT_WINDOW=60  # seconds

6. Enable MFA for All Admins
   MFA_REQUIRED_FOR_ADMINS=true

7. Set Session Timeout
   SESSION_TIMEOUT_MINUTES=30

8. Enable Audit Logging
   AUDIT_LOGGING_ENABLED=true
   AUDIT_LOG_FILE=/var/log/tecno_spark_audit.log
